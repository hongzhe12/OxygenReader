## ✨ 二氧化碳阅读器

这次改版的核心思路：**去掉反直觉的操作，让阅读器像正常窗口一样交互。**

### 1. 快捷键重设计

**之前的问题：** 4 个快捷键，每个都有硬伤。

| 键 | 问题 |
|---|---|
| `T` / `Q` | 全局热键 = 永远没法正常打字 |
| `Ctrl+Shift+Q` / `Ctrl+Shift+F` | 跟搜索等系统快捷键冲突，还要记两个组合 |

**产品思考：** 阅读器只需要三个操作——翻下、翻上、切换显示。隐藏不需要独立快捷键，因为鼠标左键点击（阅读器外部）就可以自然隐藏。所以"显示"和"隐藏"合并成一个"**切换**"键就够了。

**现在的方案：**

| 操作 | 默认键 | 理由 |
|------|--------|------|
| 下一行 | `↓` | 直觉，不干扰打字 |
| 上一行 | `↑` | 同上 |
| 切换显示/隐藏 | `F3` | 不参与打字，不和常见应用冲突 |
| 隐藏（备选） | 左键点击阅读器外部 | 最自然的 dismiss 方式 |

### 2. 拖动方式

**之前的问题：** 右键拖动阅读器。所有窗口都是用左键拖动的，右键拖动完全反肌肉记忆。原因是为了给左键让路——左键被全局监听占用了（点击即隐藏）。

**产品思考：** 不应该为了一个功能（点击隐藏）去破坏另一个更基础的功能（窗口拖动）。点击隐藏可以用 F3 替代，而窗口拖动的直觉不可妥协。

**现在的方案：** 左键=拖动，跟所有窗口一样。去掉全局鼠标监听，隐藏统一走 F3。

### 3. 控制台布局

**之前的问题：** 设置项全部摊开，占据了窗口大部分空间，书架反而缩在底部。

**产品思考：** 控制台的核心场景是"选书→开始读"。设置是低频操作，不该喧宾夺主。

**现在的方案：** 书架占主区域，⚙ 设置折叠收起。同时支持拖拽文件导入、双击书架直接阅读。

---

一句话总结：**去掉全局监听带来的各种副作用，改回标准窗口交互。**
